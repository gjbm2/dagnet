## 1. What the Chart Shows

The cohort maturity chart plots **edge conversion rate** (y/x) against
**age τ** (days since cohort anchor date) for a **group of cohorts**
selected by the user's query DSL.

The desired rendering is a single continuous curve with two visually
distinct components:

- **Evidence (e)**: what we have actually observed — solid line
- **Forecast (f)**: what we predict is still to come — shaded crown above e

The curve asymptotes at `p.mean` (the edge conversion rate), NOT at
`p∞` (the path rate). The denominator throughout is `x` (from-node
arrivals at this edge).

The eventual goal is a **fan chart** showing confidence intervals around
the evidence curve, derived from dispersion data. The e+f crown is an
interim visualisation.

---

## 2. Key Terms

### Denominator semantics

| Term | Symbol | Meaning |
|------|--------|---------|
| **From-node arrivals** | `x` | People who arrived at this edge's source node |
| **Anchor entrants** | `a` | People who entered at the cohort anchor node |
| **Edge rate** | `y/x` | Conversion rate at this specific edge |
| **Path rate** | `y/a` | Conversion rate through the full path from anchor |
| **forecast_mean** | `fm` | The edge's eventual conversion rate (= `p.forecast.mean`) |

The chart uses **edge rate** (`y/x`) throughout. `forecast_mean` is an
edge rate (~76% for a typical edge), not a path rate (~5%).

### Epochs

A group of cohorts with staggered anchor dates naturally divides
the τ axis into three epochs:

| Epoch | τ range | Description |
|-------|---------|-------------|
| **A** | `0 .. min(max_τ_i)` | All cohorts have evidence at this τ. Pure evidence, no forecast needed. |
| **B** | `min(max_τ_i) .. max(max_τ_i)` | Some cohorts have evidence, others have plateaued (their data stopped because they haven't lived long enough). Mixed e+f. |
| **C** | `> max(max_τ_i)` | No cohort has evidence at this τ. Pure forecast (synthetic frames). |

Where `max_τ_i = today - anchor_day_i` is the maximum observed age
for cohort i.

### Evidence and forecast components

- **Evidence (e)**: observed cumulative conversions `y` at this τ
- **Forecast (f)**: model-predicted additional conversions beyond what's
  been observed. `f = projected_y - evidence_y`
- **Projected**: the blended estimate = `e + f`
- **Crown**: the shaded area between the evidence line and the
  projected line on the chart

### Synthetic frames

Frames generated by the backend beyond the last real observation date.
Each synthetic frame carries:
- `y` = frozen evidence (last real observation for this cohort)
- `x` = frozen x (last real observation)
- `projected_y` = model prediction at this τ
- `forecast_y` = max(0, projected_y - y)
- `is_synthetic = true`

## 3. The Three Epochs in Detail

### 3.1 Epoch A — all evidence

Every cohort in the group has been observed at this τ. The evidence
rate Σy/Σx rises monotonically as the cohorts mature. No forecast
component needed.

### 3.2 Epoch B — mixed e+f

At the boundary between A and B, the youngest cohort drops out
(it hasn't lived long enough to have data at this τ). The key
requirement:

**The denominator Σx must be monotonically non-decreasing.**

When a cohort drops out, its last-known x continues to contribute
to the denominator. Without this, the rate jumps artificially as
the cohort mix changes (composition bias).

Implementation: for each cohort that has plateaued at a given τ,
the backend emits a carry-forward data point with:
- `y` = frozen at last observed value
- `x` = frozen at last observed value (keeps Σx stable)
- `projected_y` = model prediction at this τ (grows with CDF)
- `forecast_y` = projected_y - y (the crown component)

This way the frontend just sums all points — no special epoch logic.

The evidence line plateaus naturally (Σy stops growing for dropped
cohorts, Σx stays stable). The forecast crown grows as the model
prediction increases while evidence is frozen. The total curve
continues smoothly.

### 3.3 Epoch C — pure forecast (synthetic tail)

Beyond all real evidence. The backend generates synthetic frames
from `sweep_to + 1` to `anchor_to + t95`. Each synthetic frame
carries frozen evidence + model-predicted projected_y for all
cohorts.

**Status**: implemented in `_append_synthetic_cohort_maturity_frames`
in `api_handlers.py`. Working for some edges, not yet confirmed
for all.

## 4. Current Implementation State

### 4.1 What works

- **Epoch C synthetic frames**: the tail beyond real evidence now
  uses frozen y (evidence) and model-based projected_y. The previous
  bug where `y_future = a × fm × CDF` (mixing `a` with `x` denominator)
  is fixed. Now uses `y_future = x × fm × CDF` for the projected_y,
  with `y` = frozen evidence.
- **Evidence line in epoch A**: monotonically rising, stable
  denominator while all cohorts contribute.
- **Model CDF overlay curves**: render via span-kernel MC median, using the same MC draws the main chart uses for its midpoint. Unified single-hop and multi-hop construction in `api_handlers.py`. See [docs/current/project-bayes/51-model-curve-overlay-divergence.md](../project-bayes/51-model-curve-overlay-divergence.md) for the construction and the contract test (`graph-ops/scripts/cohort-maturity-model-parity-test.sh`) that enforces overlay-main parity.

### 4.2 What's broken / incomplete

- **Epoch B carry-forward**: not yet implemented. When cohorts drop
  out in epoch B, Σx shrinks, causing the evidence rate to jump
  artificially. Need to emit carry-forward points for plateaued
  cohorts to keep the denominator stable.
- **Blend formula in `annotate_data_point`**: the current blend
  `blended_rate = c × evidence_rate + (1-c) × fm × c` produces
  `projected ≈ evidence` when evidence tracks the model. The crown
  is invisible. This formula needs rethinking — possibly replaced
  with the pure model prediction for projected_y, or a different
  blend weight.
- **Synthetic frames not generating for some edges**: diagnostic
  logging has been added to `_append_synthetic_cohort_maturity_frames`
  but the root cause is not yet identified. Need to check whether
  the function is called and why it might exit early.
- **tauMax clipping**: the frontend clips data at tauMax = dayDiff(B,
  anchorFrom). For wide anchor ranges, this can clip synthetic frames
  for older cohorts (their τ exceeds tauMax). May need adjustment.

## 5. Remaining Work

### 5.1 Epoch B denominator stabilisation (priority 1)

Emit carry-forward data points in the backend for cohorts that have
plateaued at a given τ. This keeps Σx monotonically non-decreasing
and makes the evidence line plateau smoothly rather than jumping.

Where: either in `_append_synthetic_cohort_maturity_frames` (extend
to cover epoch B) or in a new post-processing step after real frame
annotation.

### 5.2 Forecast crown visibility (priority 2)

The blend formula in `annotate_data_point` produces projected ≈ evidence,
making the crown invisible. Options:

**A. Use pure model prediction for projected_y**: `projected_y = x × fm × CDF(τ)`.
The crown = model - evidence. Always visible when evidence is behind the
model. But the curve is not weighted by evidence (user rejected this).

**B. Use evidence extrapolation**: `projected_y = y / c` (the original
formula). Always produces projected ≥ y. Crown = y × (1/c - 1).
Visible when c < 1 (immature). But doesn't use the model at all.

**C. Fan chart (ideal)**: confidence intervals around the evidence
curve, derived from dispersion data. The proper solution. Requires
dispersion data to be available (σ of the posterior).

**D. Hybrid**: use the model as the projected curve during epoch C
(pure forecast), and evidence extrapolation during epoch B. Blends
naturally at the boundary.

### 5.3 Synthetic frame generation reliability

Need to confirm synthetic frames are generated for all edges. Current

early-return conditions. Check the Python server log after refreshing

### 5.4 tauMax alignment

Ensure the frontend tauMax extends far enough to include synthetic

## 6. Design Principles (established in this session)

1. **The FE should be very dumb.** All computation (blending, carry-
   forward, denominator stabilisation) happens in the backend. The FE
   sums and divides.

2. **Σx must be monotonically non-decreasing.** When cohorts drop out
   in epoch B, their x continues to contribute. No composition jumps.

3. **forecast_mean is an edge rate, not a path rate.** Multiplied by x,
   not a. The chart shows edge conversion rate (y/x).

4. **Synthetic frames carry frozen evidence.** Not model-predicted y.
   The evidence (y) is what was actually observed; the forecast
   (projected_y) is what the model predicts. The crown is the gap.

5. **No blend double-dampening.** The model term in the blend should
   not be multiplied by both `(1-c)` and `c`. If the model predicts
   fm × CDF(τ) at this age, the blend should weight that against
   evidence — not squash it further.

6. **The ideal visualisation is a fan chart.** The e+f crown is interim.
   When dispersion data is available, confidence intervals replace the
   binary e/f split.
